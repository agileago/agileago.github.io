webstorm显示特殊字体

发表于 2015-11-07   |  

最近一直在折腾编辑器，什么vim, emacs , sublime 了都折腾了一遍，反而没有好好看书，实在是罪大恶极啊！
因为之前一直推荐使用atom，但是发现了一些问题，于是写一篇编辑器折腾记。

前记

本人之前一直用VS开发，后来从后端转到前端，感受过VS得各种强大，才觉得前端的编辑器真的很轻量，但觉得这就应该是我想要的

Atom

好几个月我都一直在使用atom，写前端的代码，我也一直都推荐用这个编辑器，优点是真的很漂亮，漂亮的无法无天，并且它的智能提示也是
相当棒，特别是js装上atom-ternjs插件之后，简直如虎添翼，但这个编辑器后来用着用着发现如果你一直开着这个编辑器2个小时以上，或者持续不断的
编码1小时以上，会发现他越来越慢，无论做什么操作都是相当慢，这个时候你必须把他重启一下，这个特点真的让我很捉急，但是假如你只是写很少量的前端
代码，这个编辑器是你的不二选择.

Sublime

就是因为atom很慢，我才试了一试大部分码农都在用的，说实话，比atom快了10倍以上，差一点我就换成他了，但折腾了一天之后发现，他总是缺少点什么东西
，css智能提示不太好，js更不用说了，也比不上atom，当我把一些在atom上漂亮的主题换到sublime上的时候，发现变了个样子，不是特别好看，并且他还不支持
ES6语法高亮，这个真的很蛋疼，sublime还有个静态提示功能，我觉得这个功能纯属扯淡，一点都不精确，每次敲代码的时候都给我弹出来，真是恶心死我了，
写什么总感觉离完美差点距离。于是它就从我的Dock中消失了

Vim

折腾前两个编辑器的时候，我就会去知乎上搜相关的编辑器推荐以及评价，但发现很多人都在推荐Vim,说什么它是神之编辑器，本着一颗好奇心，就在mac上
装上一瞧真面目，它是直接shell上编辑的,这个编辑器上手真的很复杂，要记各种快捷键，也没提示，也没高亮，这些东西都要自己摸索着去配置，简直难用之及，
真不明白一堆人在那里吹这个编辑器有多好用，我觉得是这些人折腾了几个月，终于把这个编辑器学的差不多了，但他们不想对新手说这个编辑器有多难用，那样体现不了
他们的逼格，建议新手心里脆弱的话千万别去这玩意，简直就是摧残人，有这时间还不如好好学一门语言来的实在，新手勿进。

Webstorm

这个是我编辑器折腾记最终选择的，其实这个东西我有一段时间学习GO语言的时候用的，但我是一个非常注重编辑器美观的患者，当时WS给我的印象就是智能提示
真的很强大，但外观我就呵呵了，没见过这么丑的，后来公司也没人用GO，没地方实践，也就不了了之了，然后转到了前段开发，但在试了这么多编辑器后，一直没有一个
相当满意的，直到最近听说webstorm发布了11，听说改进挺大，然后赶紧下载下来看看，突然发现他的字体渲染变好了，变得相当好看了，就开始折腾他了，
发现在mac上的速度还是相当可以的，然后配了one dark的配色，再配上Source Code Pro Italic的字体，还是相当好看的，当然他的图标还是有待改进，
但我经过一轮折腾后终于找到了自己的选择，也是挺好的。说一个webstorm上设置特殊字体的小技巧：

在mac上你下载完 Source Code Pro 字体并且安装完后，发现这个字体下面有一大堆字体，但在webstorm字体设置那块却只出现一个Source Code Pro的字体，
选完之后发现用得是Regular的字体，但我最喜欢Source Code Pro Italic的字体，这个时候怎么办呢? 你把这个字体文件夹里面的其他字体都给停用掉，
只剩下你想用的那个字体，这个时候在选择字体，他就会应用上你想要的字体

总结

折腾一圈，确定为平时一些小项目，就用Atom来做，而像稍微大点得项目，比如node什么之类的就用webstorm来做，哎呀，编辑器之旅就结束吧，还是多看点书
多积累点技术，提高自己最重要，这些东西对自己的加成还是很少的。拜拜了，编辑器之争！
